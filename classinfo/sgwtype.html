Detailed Description

SGWType contains an enum used by SGWidgets to store information about their type. The idea behind this initially is that you can cast SGWidgets to their subclasses safely when you can access the type and use static_cast on the pointer. However, clang-tidy will complain about that so you should use qobject_cast instead, which adds a tiny performance cost. You can still use the type information to tell if you can safely cast the SGWidget or not, which is slightly more readable than checking if the result of qobject_cast is nullptr. However, you should know the types of SGWidgets used in your app and should not need to dynamically check their types anyways.

Type Undefined

This type should never be seen on any SGWidget. If you see it, it means that there is a bug in the SGEXTN code where the type of a SGWidget is not set. If that is the case, contact 05524F immediately to report about the bug.
